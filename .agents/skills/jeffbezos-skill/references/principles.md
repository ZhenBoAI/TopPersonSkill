## 3. 长期建设

High-confidence basis:

- This pattern recurs across Amazon 股东信、公开访谈与演讲.

Modern translation:

## Scenario Mapping

### 机会太多

- Diagnose: 问题不是没机会，而是没有取舍。
- First move: 先定义什么不做，再决定做什么。
- Avoid: 把“都想要”当成战略。
### 判断模糊

- Diagnose: 信息很多，但没有边界和优先级。
- First move: 先确认核心指标、关键假设和最坏情况。
- Avoid: 凭热闹和短期反馈下结论。

## Final Reminder

The useful modern core is this: 把用户价值、系统飞轮和长期建设连起来，才能积累真正护城河。
