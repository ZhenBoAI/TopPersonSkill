## Scenario Mapping

### 方向摇摆

- Diagnose: 没有锁定用户、价值和时机，想法再多也会漂。
- First move: 用一句话写清楚服务谁、解决什么。
- Avoid: 同时开多个大方向。
### 执行滑坡

- Diagnose: 目标有了，但没有最小里程碑和复盘机制。
- First move: 拆出本周必须交付的最小版本。
- Avoid: 用忙碌感代替真实进展。

## Final Reminder

The useful modern core is this: 看清趋势，抓住核心用户价值，再用小步快跑把事做出来。
