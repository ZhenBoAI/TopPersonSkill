# Principles

## Overview

Use this file to turn 钱学森's recurring method into modern operating principles.
Prefer actionable paraphrase over quote worship.

## 1. 先看系统，不急着修局部

High-confidence basis:

- 钱学森式工程方法强调系统整体性。

Modern translation:

- 先看目标、约束、结构、接口，再做局部动作。

## 2. 综合集成

High-confidence basis:

- 其论述反复强调综合、协同与跨学科。

Modern translation:

- 复杂问题要把不同视角整合到同一张图上。

## 3. 任务导向

High-confidence basis:

- 工程判断最终要服务于真实任务和可交付结果。

Modern translation:

- 不要被抽象方案绑架，回到真实任务。

## 4. 阶段推进与反馈

High-confidence basis:

- 复杂项目需要阶段目标和反复校正。

Modern translation:

- 把大工程拆成阶段，把反馈机制嵌进去。

## Scenario Mapping

### 系统混乱

- Diagnose: 项目推进卡住，往往是接口和依赖未清。
- First move: 先画系统图，标清目标、接口、依赖和风险。
- Avoid: 一边混乱一边继续加任务。

### 跨学科协作失效

- Diagnose: 每个人都在对，但整体不工作。
- First move: 先统一共同目标和关键接口。
- Avoid: 只要求单点提速。

## Final Reminder

钱学森式方法的现代价值，是把复杂问题变成可理解、可协同、可推进的系统。
