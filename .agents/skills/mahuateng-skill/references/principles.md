# Principles

## Overview

Use this file to turn 马化腾's recurring method into modern operating principles.
Prefer actionable paraphrase over quote worship.

## 1. 用户动作先于功能堆叠

High-confidence basis:

- 马化腾式产品判断更看真实用户动作。

Modern translation:

- 先把最常用、最重要的动作做顺。

## 2. 节奏比热闹更重要

High-confidence basis:

- 平台型产品常靠时机和节奏获胜。

Modern translation:

- 别在热闹时追着跑，要看自己该不该在此刻出手。

## 3. 平台视角

High-confidence basis:

- 产品决策不能只看单点，要看系统位置。

Modern translation:

- 判断一个功能时，也要看它如何进入整体结构。

## 4. 稳住核心再扩展

High-confidence basis:

- 长期平台建设依赖稳定核心。

Modern translation:

- 先稳住关键链路，再做扩张。

## Scenario Mapping

### 功能膨胀

- Diagnose: 产品重、慢、乱，往往是加得太快。
- First move: 先砍掉低频、低价值、打断主路径的功能。
- Avoid: 继续往上叠功能。

### 时机摇摆

- Diagnose: 外部声音太多，内部判断失焦。
- First move: 先看用户行为和自身条件，再决定是否进入。
- Avoid: 被行业热点带着冲动决策。

## Final Reminder

马化腾式方法的价值，在于克制、节奏感和平台视角，而不是跟风快。
