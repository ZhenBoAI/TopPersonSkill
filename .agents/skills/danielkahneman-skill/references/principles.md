# Principles

## Overview

Use this file to turn 丹尼尔·卡尼曼's recurring method into modern operating principles.
Prefer actionable paraphrase over quote worship.

## 1. 先修决策流程

High-confidence basis:

- Kahneman 的价值不只是识别偏差，更是改善流程。

Modern translation:

- 先让决策过程更干净，再讨论谁更有感觉。

## 2. 看基准率

High-confidence basis:

- 个案故事常会误导判断。

Modern translation:

- 重大判断时，先看类似问题的大样本基准。

## 3. 识别噪音

High-confidence basis:

- 同类判断差异过大，说明流程有噪音。

Modern translation:

- 先统一标准和输入，再让人判断。

## 4. 慢下来检查框架

High-confidence basis:

- 很多错误来自问题被错误定义。

Modern translation:

- 先确认你在回答的是不是正确问题。

## Scenario Mapping

### 拍板太快

- Diagnose: 时间压力一来，直觉会过度接管。
- First move: 先加一个短暂停顿，补基准率和备选项。
- Avoid: 用“经验丰富”直接跳过校验。

### 判断分歧过大

- Diagnose: 同样材料下结论差太多，往往是流程不稳。
- First move: 先统一标准和评估维度。
- Avoid: 继续靠更大声争赢。

## Final Reminder

Kahneman 式方法的关键，不是背概念，而是让判断过程更干净、更稳。
